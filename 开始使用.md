### 方案 B：部署到 GitHub + Streamlit Cloud

这是让全世界都能访问你网站的方法！

#### 第 1 步：创建 GitHub 仓库

1. 访问 https://github.com
2. 点击右上角 `+` → `New repository`
3. 填写信息：
   - Repository name: `wiper-size-query` （或你喜欢的名字）
   - 选择 **Public**（公开）
   - **不要**勾选 "Initialize with README"
4. 点击 `Create repository`

#### 第 2 步：推送代码到 GitHub

在项目目录打开命令行，执行：

```bash
# 初始化 Git
git init

# 添加所有文件
git add .

# 提交
git commit -m "雨刷尺寸查询系统"

# 设置主分支
git branch -M main

# 添加远程仓库（替换成你的 GitHub 地址）
git remote add origin https://github.com/你的用户名/wiper-size-query.git

# 推送到 GitHub
git push -u origin main
```

**遇到权限问题？**
- 可能需要配置 GitHub Personal Access Token
- 或者配置 SSH 密钥
- 参考：https://docs.github.com/cn/authentication

#### 第 3 步：在 Streamlit Cloud 部署

1. **访问 Streamlit Cloud**
   - 打开：https://share.streamlit.io
   - 用 GitHub 账号登录

2. **创建新应用**
   - 点击 `New app` 按钮

3. **配置**
   - Repository: 选择你刚创建的仓库
   - Branch: `main`
   - Main file path: `app.py`

4. **部署**
   - 点击 `Deploy!`
   - 等待 2-5 分钟

5. **完成！**
   - 你会得到一个公开 URL
   - 例如：`https://你的用户名-wiper-size-query-xxxxx.streamlit.app`
   - 分享这个链接给任何人！

## 💡 使用提示

1. **搜索功能**
   - 输入车型部分名称即可
   - 例如：输入 "A" 会匹配 A4、A6
   - 输入 "美瑞" 会匹配 凯美瑞

2. **数据格式**
   - 确保 Excel 列名不变
   - 空值用 "-" 表示
   - 年份格式：2020-2023

3. **部署更新**
   - 任何修改推送到 GitHub 后
   - Streamlit Cloud 会自动更新
   - 通常 1-2 分钟完成

## ❓ 常见问题

**Q1: 本地运行时找不到模块？**
```bash
python -m pip install -r requirements.txt
```

**Q2: 数据文件读取失败？**
- 确保 wiper_data.xlsx 在项目根目录
- 检查文件名大小写

**Q3: 部署后看不到数据？**
- 确保数据文件已推送到 GitHub
```bash
git add wiper_data.xlsx
git commit -m "添加数据文件"
git push
```

**Q4: 中文显示乱码？**
- 确保所有文件使用 UTF-8 编码保存
- Excel 另存为时选择 UTF-8

**Q5: 如何自定义界面样式？**
- 编辑 `app.py` 中的 CSS 部分
- 或修改 `.streamlit/config.toml` 主题配置
